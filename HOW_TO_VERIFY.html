How to Verify a PLRS Record

This page explains how any person can independently verify a Public Legal Record System (PLRS) Event or Challenge using only:

  • Publicly downloaded files
  • Open verification tools
  • Basic command-line access
Important: You do NOT need to trust the website hosting these files. Verification is based on cryptographic math, not institutional authority.

1. What You Need to Download

From the public mirror, download:
  • public_index.json
  • public_challenge_index.json (if challenges exist)
  • The specific Event file: E###.json
  • The specific Challenge file: C###.json (if applicable)
  • The PLRS verification tools package (or source scripts)

2. Verify a Single Event

Run: 
python validate_event.py E123.json
python hash_event.py E123.json
You should receive a SHA-256 hash. Then confirm that the same hash appears in: 
public_index.json
If they match:
✅ The Event is authentic and unmodified.

3. Verify a Challenge

Run: 
python validate_challenge.py C45.json
python hash_challenge.py C45.json
Confirm the resulting hash appears in: 
public_challenge_index.json
Also confirm that: 
Challenged_Event_ID
exists in: 
public_index.json
If all match:
✅ The Challenge is authentic and properly linked.

4. Verify a Court Evidence Bundle

Run: 
python verify_bundle.py PLRS_EventBundle_E123.json
Or: 
python verify_bundle.py PLRS_ChallengeBundle_C45.json
If it passes:
✅ The bundle matches both the record and the ledger.

5. Verify Chain of Custody

Run: 
python verify_chain_of_custody.py PLRS_ChainOfCustody_E123.json
This confirms:
  • Strict time ordering
  • No backdating
  • No broken ownership transfers

6. Verify the Entire Ledger

Run: 
python verify_ledger_integrity.py \
  --event-index public_index.json \
  --challenge-index public_challenge_index.json \
  --events-dir events \
  --challenges-dir challenges
If this passes:
✅ The entire PLRS dataset is internally consistent and untampered.

7. What Verification Proves

Successful verification proves:
  • The record has not been edited
  • The timeline has not been altered
  • The ledger has not been pruned
  • Challenges cannot overwrite history
  • Public mirrors cannot falsify records

8. What Verification Does NOT Prove

PLRS verification does NOT determine:
  • Who is morally right
  • Who legally prevails
  • Whether testimony is true
It only proves whether records were altered.

9. Why This Matters

In traditional systems:
  • Records change silently
  • Timelines shift
  • Reports overwrite each other
  • The public cannot verify anything
In PLRS:
  • Every change becomes visible forever
  • Challenges append instead of replace
  • All integrity is independently provable

10. Final Statement

You do not need permission to verify.

You do not need to trust the publisher.

If the hashes match, the record is real.
